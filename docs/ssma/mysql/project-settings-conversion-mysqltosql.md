# <a name="project-settings-conversion-mysqltosql"></a>项目设置（转换）(MySQLToSQL)
"**项目设置**" 对话框的 "转换" 页包含用于自定义 SSMA 将 MySQL 语法转换为 SQL Server 或 SQL Azure 语法的方式的设置。  
  
"**项目设置**" 和 "**默认项目设置**" 对话框中提供了 "转换" 窗格。  
  
-   使用 "**默认项目设置**" 对话框可以为所有项目设置配置选项。 若要访问转换设置，请在 "**工具**" 菜单上，选择 "**默认项目设置**"，从 "**迁移目标版本**" 下拉框中选择需要查看其设置的 "迁移项目类型"，单击左侧窗格底部的 "**常规**"，然后选择 "**转换**"。  
  
-   若要指定当前项目的设置，请在 "**工具**" 菜单上单击 "**项目设置**"，然后单击左窗格底部的 "**常规**"，然后单击 "**转换**"。  
  
## <a name="options"></a>选项  
  
### <a name="collate-clause"></a>Collate 子句  
  
|||  
|-|-|  
|**术语**|**定义**|  
|**显式排序规则转换**|显式 COLLATE 子句转换选项指定如何转换 MySQL 代码中的显式 COLLATE 子句。 可能的选择：忽略并标记为警告/生成错误<br /><br />**默认模式**：忽略并使用警告标记<br /><br />**乐观模式**：忽略并使用警告标记<br /><br />**完全模式**：忽略并使用警告标记|  
  
### <a name="column-constraints"></a>列约束  
  
|||  
|-|-|  
|**术语**|**定义**|  
|**为枚举数据类型的列生成约束**|为 SQL Server 或 SQL Azure 表中 ENUM 数据类型的列生成约束（如果它不存在于 MySQL 表中）。 如果为 "是"，则枚举数据类型的所有已转换列都将附带控制值的 CHECK 约束。<br /><br />**默认模式**：否<br /><br />**乐观模式**：否<br /><br />**完整模式**：是|

|**为无符号数值数据类型列的列生成约束**|将非负值的值添加到无符号数值数据类型的列。<br /><br />**默认模式**：否<br /><br />**乐观模式**：否<br /><br />**完整模式**：是|  
|**为 YEAR 数据类型列生成约束**|为 SQL Server 或 SQL Azure 表中的年份数据类型列生成约束（如果它不存在于 MySQL 表中）。 如果为 "是"，则年份数据类型的所有已转换列都将附带控制该值的 CHECK 约束。<br /><br />**默认模式**：否<br /><br />**乐观模式**：否<br /><br />**完整模式**：是|
